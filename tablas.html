Etiquetas de Tablas

Las tablas permiten mostrar información organizada en filas y columnas. Son ideales para datos comparativos, horarios y reportes. Nunca deben usarse para maquetar el diseño de una página.

<table>

Definición: Es el contenedor principal de una tabla. Por sí sola no muestra nada; necesita las etiquetas tr, th y td para mostrar datos.

Sintaxis básica:

<table>
  <caption>Tabla de notas</caption>
  <thead> ... </thead>
  <tbody> ... </tbody>
  <tfoot> ... </tfoot>
</table>

Uso adecuado: Para mostrar datos comparativos, calendarios, horarios o cualquier información con relación entre filas y columnas.

Buenas prácticas:

  • Siempre agregar caption para describir el contenido de la tabla.
  • Nunca usar tables para maquetar páginas; usar CSS Grid o Flexbox.
  • Separar la tabla en thead, tbody y tfoot para mayor claridad semántica.
<tr>, <th> y <td>

Definición: tr define una fila dentro de la tabla. th define una celda de encabezado (negrita y centrada por defecto). td define una celda de datos normal. Las tres trabajan siempre juntas para construir la estructura interna de la tabla.

Sintaxis básica:

<table>
  <thead>
    <tr>
      <th scope="col">Nombre</th>
      <th scope="col">Nota</th>
      <th scope="col">Estado</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana García</td>
      <td>4.5</td>
      <td>Aprobado</td>
    </tr>
    <tr>
      <td>Luis Pérez</td>
      <td>3.8</td>
      <td>Aprobado</td>
    </tr>
  </tbody>
</table>

Uso adecuado: Usar colspan para fusionar celdas horizontalmente y rowspan para fusionar celdas verticalmente cuando los datos lo requieran.

Buenas prácticas:

  • Usar scope="col" o scope="row" en th para mejorar la accesibilidad.
  • No aplicar estilos directamente en los atributos de td; usar clases CSS.
  • Incluir thead, tbody y opcionalmente tfoot para estructura semántica.
<caption>

Definición: Define el título o descripción de una tabla. Va justo después de la etiqueta de apertura table y se muestra centrado encima de la tabla por defecto.

Sintaxis básica:

<table>
  <caption>Listado de estudiantes y notas</caption>
  ...
</table>

Uso adecuado: Fundamental para accesibilidad. Los lectores de pantalla lo anuncian antes de leer la tabla para que el usuario sepa qué información va a escuchar.

Buenas prácticas:

  • Siempre incluirlo en tablas de datos.
  • Escribir una descripción breve pero informativa.
  • No usarlo como reemplazo de un h2 o h3 fuera de la tabla.
