Etiquetas de Texto

Las etiquetas de texto permiten organizar y dar significado al contenido escrito. Indican al navegador y buscadores qué partes son más importantes, cuáles son títulos y cuáles son párrafos normales.

<h1> - <h6>

Definición: Etiquetas de encabezado que definen la jerarquía de títulos. h1 es el más importante y h6 el menos.

Sintaxis básica:

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

Uso adecuado: Representan estructura lógica, no tamaños de letra. Si quieres cambiar el tamaño visual usa CSS, no cambies de nivel de encabezado.

Buenas prácticas:

  • Usar solo un h1 por página.
  • No saltar niveles: no ir de h1 directamente a h4.
  • No usarlos solo para hacer el texto más grande.
<p>

Definición: Define un párrafo de texto. Es la etiqueta de texto más usada en HTML. El navegador agrega espacio antes y después automáticamente.

Sintaxis básica:

<p>Este es un párrafo de ejemplo.</p>

Uso adecuado: Usar para agrupar oraciones relacionadas que forman una idea completa.

Buenas prácticas:

  • No usar br varias veces seguidas para separar textos; para eso sirve p.
  • No meter etiquetas de bloque como div o h2 dentro de un p.
<strong> y <em>

Definición: strong marca texto con importancia semántica fuerte (negrita). em marca texto con énfasis (cursiva). Ambas tienen significado real para los lectores de pantalla.

Sintaxis básica:

<p>Es <strong>obligatorio</strong> guardar
antes de <em>cerrar el editor</em>.</p>

Uso adecuado: Usar cuando el contenido realmente requiere énfasis semántico, no solo visual.

Buenas prácticas:

  • No usarlas solo para cambiar el aspecto visual del texto; para eso está CSS.
  • No abusar: si todo está marcado como importante, nada lo es realmente.
<span>

Definición: Contenedor en línea sin significado semántico propio. Se usa para aplicar estilos CSS o identificar con JavaScript una parte específica del texto.

Sintaxis básica:

<p>El descuento es del
<span class="resaltado">30%</span> hoy.</p>

Uso adecuado: Ideal cuando no existe ninguna etiqueta semántica más adecuada para marcar un fragmento de texto en línea.

Buenas prácticas:

  • Preferir etiquetas semánticas (strong, em, abbr) cuando aplique.
  • Siempre usar junto con una clase o id para darle propósito claro.
