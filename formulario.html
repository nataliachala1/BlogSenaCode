4. Formularios

Los formularios permiten recopilar información del usuario. Son fundamentales en páginas de registro, contacto y compras en línea.

<form>

Definición: Contenedor principal de un formulario. Define hacia dónde se envían los datos y el método de envío.

Sintaxis básica:

<form action="/enviar" method="post">
  <!-- campos aquí -->
</form>

Buenas prácticas:

  • Usar method="post" para datos privados como contraseñas.
  • No anidar un form dentro de otro form.
<input>

Definición: Campo de entrada más versátil. Cambia según el atributo type: texto, contraseña, email, checkbox, etc.

Sintaxis básica:

<input type="text" name="nombre" placeholder="Tu nombre" required>
<input type="email" name="correo">
<input type="password" name="clave">
<input type="checkbox" name="terminos">

Buenas prácticas:

  • Siempre incluir el atributo name.
  • Usar required en campos obligatorios.
<label>

Definición: Etiqueta descriptiva de un campo. Al hacer clic en ella, el cursor se mueve al campo asociado.

Sintaxis básica:

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre">

Buenas prácticas:

  • El valor de for debe ser igual al id del input.
  • Nunca omitir el label.
<select>, <textarea> y <button>

Definición: select crea un menú desplegable. textarea permite texto multilínea. button crea un botón interactivo.

Sintaxis básica:

<select name="ciudad">
  <option value="">Selecciona una ciudad</option>
  <option value="bogota">Bogotá</option>
</select>

<textarea name="mensaje" rows="5"
  placeholder="Escribe aquí..."></textarea>

<button type="submit">Enviar</button>

Buenas prácticas:

  • Siempre especificar type en button.
  • Agregar una primera opción vacía en select.
