Etiquetas Multimedia

Las etiquetas multimedia permiten integrar imágenes, videos y audios directamente en la página sin necesidad de plugins externos. Desde HTML5 el navegador los reproduce de forma nativa.

<img>

Definición: Inserta una imagen en el documento. Es una etiqueta vacía; la imagen se carga desde la ruta indicada en el atributo src. El atributo alt es obligatorio.

Sintaxis básica:

<img src="foto.jpg"
     alt="Descripción de la imagen"
     width="300" height="300">

Ejemplo visual:

Uso adecuado: Para imágenes que forman parte del contenido. Si la imagen es decorativa usar alt="" vacío para que los lectores de pantalla la ignoren.

Buenas prácticas:

  • Nunca omitir el atributo alt; su ausencia es un error de accesibilidad.
  • Especificar width y height para evitar saltos al cargar.
  • Usar loading="lazy" en imágenes lejos del inicio de la página.
  • Usar formatos modernos como WebP para reducir el peso del archivo.
<video>

Definición: Incrusta un reproductor de video nativo directamente en la página. Admite múltiples fuentes con la etiqueta source para garantizar compatibilidad entre navegadores.

Sintaxis básica:

<video controls width="640" height="360">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <p>Tu navegador no soporta video HTML5.</p>
</video>

Ejemplo visual:

Uso adecuado: El texto entre las etiquetas sirve de mensaje alternativo para navegadores muy antiguos que no soporten HTML5.

Buenas prácticas:

  • No usar autoplay con sonido; los navegadores modernos lo bloquean.
  • Agregar subtítulos con la etiqueta track para accesibilidad.
  • Usar preload="metadata" para cargar solo la información básica al inicio.
<audio>

Definición: Incrusta un reproductor de audio nativo en la página. Funciona de manera similar a video pero solo para contenido de sonido.

Sintaxis básica:

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Tu navegador no soporta audio HTML5.
</audio>

Ejemplo visual:

Uso adecuado: Podcasts, efectos de sonido o música de fondo.

Buenas prácticas:

  • Nunca reproducir audio automáticamente sin que el usuario lo solicite.
  • Incluir siempre el atributo controls para que el usuario pueda pausar.
  • Ofrecer al menos dos formatos (mp3 y ogg) para mayor compatibilidad.
<iframe>

Definición: Permite incrustar contenido externo dentro de la página, como videos de YouTube, mapas de Google Maps o documentos PDF, en un marco independiente.

Sintaxis básica:

<iframe
  src="https://www.youtube.com/embed/ID"
  width="560" height="315"
  title="Descripción del video"
  allowfullscreen>
</iframe>

Uso adecuado: Para embeber contenido externo sin redirigir al usuario a otro sitio.

Buenas prácticas:

  • Siempre incluir el atributo title con una descripción del contenido.
  • Usar sandbox para limitar los permisos del contenido externo.
  • No intentar incrustar páginas que no lo permitan.
