Blog Educativo: Etiquetas HTML por Categorías

Este blog es una guía educativa sobre las etiquetas más importantes de HTML. Está organizado en seis categorías: estructurales, texto, listas, formularios, tablas y multimedia. Para cada etiqueta encontrarás su definición, un ejemplo de uso y buenas prácticas recomendadas.

HTML (HyperText Markup Language) es el lenguaje base de todas las páginas web. Fue creado por Tim Berners-Lee en 1991 y desde entonces ha evolucionado hasta su versión actual, HTML5. Aprender sus etiquetas es el primer paso para construir sitios web.

1. Etiquetas Estructurales

Las etiquetas estructurales definen el esqueleto de una página web. Sin ellas, el navegador no sabría cómo interpretar el documento. Son las primeras que debes aprender porque forman la base de cualquier archivo HTML.

<html>

Definición: Es la etiqueta raíz de todo documento HTML. Todo el contenido de la página debe estar dentro de ella. Le indica al navegador que el archivo está escrito en HTML.

Sintaxis básica:

<html lang="es">
  ...
</html>

Ejemplo práctico:

<!DOCTYPE html>
<html lang="es">
  <head><title>Mi página</title></head>
  <body><p>Hola mundo</p></body>
</html>

Uso adecuado: Debe aparecer justo después del DOCTYPE. Solo puede haber una etiqueta html por documento y debe contener únicamente head y body como hijos directos.

Buenas prácticas:

  • Siempre escribir el atributo lang con el idioma de la página (es, en, fr, etc.).
  • No poner contenido directamente dentro de html sin usar head o body.
  • No omitir el DOCTYPE antes de esta etiqueta.

<head>

Definición: Contiene la información del documento que no se muestra directamente en pantalla. Dentro van el título de la página, los estilos CSS, los scripts y los metadatos como el charset o la descripción para buscadores.

Sintaxis básica:

<head>
  <meta charset="UTF-8">
  <title>Título de la página</title>
</head>

Uso adecuado: Va siempre antes del body. Todo lo que se ponga aquí afecta el comportamiento o apariencia de la página pero no es visible para el usuario directamente.

Buenas prácticas:

  • Incluir siempre meta charset="UTF-8" para evitar problemas con tildes y caracteres especiales.
  • Escribir un title descriptivo y único en cada página.
  • Agregar la etiqueta meta viewport para que la página se vea bien en celulares.

<body>

Definición: Contiene todo el contenido visible de la página web: textos, imágenes, videos, tablas, formularios, botones, etc. Todo lo que el usuario ve en el navegador debe estar dentro del body.

Sintaxis básica:

<body>
  <h1>Bienvenido</h1>
  <p>Este es el contenido de mi página.</p>
</body>

Uso adecuado: Solo puede existir un body por documento. Es el segundo hijo directo de html, después de head.

Buenas prácticas:

  • No usar atributos visuales en línea como bgcolor o text; usar CSS para eso.
  • Organizar el contenido con etiquetas semánticas como header, main, section y footer.

<header>

Definición: Representa la cabecera de una página o sección. Generalmente contiene el logo del sitio, el título principal y la navegación. Ayuda a los lectores de pantalla y buscadores a identificar la parte introductoria del contenido.

Sintaxis básica:

<header>
  <h1>Mi Sitio Web</h1>
  <nav>...</nav>
</header>

Buenas prácticas:

  • No confundir con la etiqueta head; son completamente distintas.
  • No anidar un header dentro de otro header.

<main>

Definición: Delimita el contenido principal de la página. Solo debe haber uno por documento. Es muy útil para la accesibilidad porque permite a los lectores de pantalla saltar directamente al contenido importante.

Sintaxis básica:

<main>
  <h2>Artículo principal</h2>
  <p>Contenido más importante de la página.</p>
</main>

Buenas prácticas:

  • Usar solo un main por página.
  • No meterlo dentro de header, footer, nav o aside.

<footer>

Definición: Define el pie de página de un documento o sección. Suele contener información de derechos de autor, enlaces legales, datos de contacto o créditos del autor.

Sintaxis básica:

<footer>
  <p>&copy; 2025 Mi Sitio. Todos los derechos reservados.</p>
</footer>

Buenas prácticas:

  • Mantenerlo simple y con información relevante.
  • Se puede usar también dentro de un article para indicar el autor y la fecha.

2. Etiquetas de Texto

Las etiquetas de texto permiten organizar y dar significado al contenido escrito de una página. No solo sirven para mostrar palabras; también indican al navegador y a los buscadores qué partes del texto son más importantes, cuáles son títulos y cuáles son párrafos normales.

<h1> - <h6>

Definición: Son las etiquetas de encabezado. Definen la jerarquía de los títulos en la página. h1 es el más importante (título principal) y h6 es el menos importante. Los buscadores como Google los usan para entender de qué trata la página.

Sintaxis básica:

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

Uso adecuado: Representan estructura lógica, no tamaños de letra. Si quieres cambiar el tamaño visual de un título, usa CSS, no cambies de nivel de encabezado.

Buenas prácticas:

  • Usar solo un h1 por página.
  • No saltar niveles: no ir de h1 directamente a h4.
  • No usar encabezados solo para hacer el texto más grande.

<p>

Definición: Define un párrafo de texto. Es la etiqueta de texto más usada en HTML. El navegador agrega automáticamente espacio antes y después de cada párrafo para separarlo visualmente del resto del contenido.

Sintaxis básica:

<p>Este es un párrafo de ejemplo con información relevante.</p>

Uso adecuado: Usar para agrupar oraciones relacionadas que forman una idea completa.

Buenas prácticas:

  • No usar la etiqueta br varias veces seguidas para separar textos; para eso sirve p.
  • No meter etiquetas de bloque como div o h2 dentro de un p.

<strong> y <em>

Definición: strong marca texto con importancia semántica fuerte; el navegador lo muestra en negrita. em marca texto con énfasis; el navegador lo muestra en cursiva. Ambas etiquetas tienen significado real para los lectores de pantalla, que cambian el tono de voz al leerlas.

Sintaxis básica:

<p>Es <strong>obligatorio</strong> guardar el archivo
antes de <em>cerrar el editor</em>.</p>

Buenas prácticas:

  • No usarlas solo para cambiar el aspecto visual del texto; para eso está CSS.
  • No abusar de ellas: si todo está marcado como importante, nada lo es realmente.

<span>

Definición: Es un contenedor en línea sin significado semántico propio. Se usa para aplicar estilos CSS o identificar con JavaScript una parte específica del texto dentro de un párrafo u otra etiqueta.

Sintaxis básica:

<p>El descuento es del <span class="resaltado">30%</span> hoy.</p>

Buenas prácticas:

  • Preferir etiquetas con significado (strong, em, abbr) cuando aplique.
  • Siempre usar span junto con una clase o id para darle propósito claro.

3. Listas

Las listas permiten organizar información en grupos de elementos relacionados. En HTML existen tres tipos: listas desordenadas (viñetas), listas ordenadas (numeradas) y listas de definición (término con descripción). Son muy usadas en menús de navegación, pasos de instrucciones y glosarios.

<ul> - Lista desordenada

Definición: Crea una lista de elementos sin orden específico. Cada elemento se muestra con una viñeta por defecto. El orden en que aparecen los ítems no es importante para el significado.

Sintaxis básica:

<ul>
  <li>Manzana</li>
  <li>Naranja</li>
  <li>Pera</li>
</ul>

Uso adecuado: Listas de ingredientes, características de un producto, opciones de menú de navegación, cualquier conjunto de elementos donde el orden no cambia el significado.

Buenas prácticas:

  • El único hijo directo válido de ul es li.
  • Cambiar el estilo de las viñetas con CSS (list-style-type) en lugar de hacks visuales.

<ol> - Lista ordenada

Definición: Crea una lista de elementos numerados automáticamente. El orden de los ítems es importante y tiene significado. Por defecto usa números (1, 2, 3...) pero se puede cambiar a letras o números romanos con CSS.

Sintaxis básica:

<ol>
  <li>Abrir el navegador</li>
  <li>Escribir la dirección web</li>
  <li>Presionar Enter</li>
</ol>

Uso adecuado: Instrucciones paso a paso, recetas, rankings o cualquier secuencia donde el orden cambia el resultado.

Buenas prácticas:

  • Usar el atributo start para comenzar desde un número diferente al 1.
  • Usar el atributo reversed para contar en orden descendente.

<dl>, <dt>, <dd> - Lista de definición

Definición: dl crea una lista de términos con sus descripciones. dt define el término y dd define su descripción. Es perfecta para glosarios, diccionarios o cualquier lista de pares clave-valor.

Sintaxis básica:

<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado para crear páginas web.</dd>
  <dt>CSS</dt>
  <dd>Lenguaje para dar estilo visual a las páginas.</dd>
</dl>

Buenas prácticas:

  • Un dt puede tener varios dd asociados si el término tiene varias acepciones.
  • No usar dl solo para crear sangría visual; usar CSS para eso.

4. Formularios

Los formularios permiten recopilar información del usuario: nombres, correos, contraseñas, opciones, archivos, etc. Son fundamentales en páginas de registro, inicio de sesión, contacto y compras en línea. Un formulario bien construido mejora la experiencia del usuario y facilita el procesamiento de datos en el servidor.

<form>

Definición: Es el contenedor principal de un formulario. Define hacia dónde se envían los datos (atributo action) y el método de envío (atributo method: get o post). Todos los campos deben ir dentro de esta etiqueta.

Sintaxis básica:

<form action="/enviar-datos" method="post">
  <!-- campos aquí -->
</form>

Uso adecuado: Usar method="post" para enviar datos privados como contraseñas. Usar method="get" solo para búsquedas donde los datos pueden aparecer en la URL.

Buenas prácticas:

  • Siempre asociar cada campo con un label.
  • No anidar un form dentro de otro form.
  • Usar el atributo novalidate solo cuando se implementa validación propia con JavaScript.

<input>

Definición: Es el campo de entrada más versátil del formulario. Cambia su comportamiento según el valor del atributo type. Puede ser texto, contraseña, checkbox, radio, fecha, correo, archivo, botón de envío y muchos más tipos.

Sintaxis básica:

<input type="text" name="nombre" placeholder="Tu nombre" required>
<input type="email" name="correo" placeholder="tucorreo@email.com">
<input type="password" name="clave">
<input type="checkbox" name="terminos" value="aceptado">
<input type="submit" value="Enviar">

Buenas prácticas:

  • Siempre incluir el atributo name para que el dato llegue al servidor.
  • Usar el type correcto ya que activa validaciones automáticas del navegador.
  • Usar required en campos obligatorios.

<label>

Definición: Es la etiqueta descriptiva de un campo. Indica al usuario qué información debe escribir. Al hacer clic en el label, el cursor se mueve automáticamente al campo asociado. Es fundamental para la accesibilidad.

Sintaxis básica:

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre">

Buenas prácticas:

  • El valor del atributo for en el label debe ser igual al id del input.
  • Nunca omitir el label; si no debe verse, ocultarlo con CSS.

<select>, <textarea> y <button>

Definición: select crea una lista desplegable de opciones. textarea permite escribir texto en múltiples líneas. button crea un botón que puede enviar el formulario, limpiarlo o ejecutar una acción con JavaScript.

Sintaxis básica:

<select name="ciudad">
  <option value="">Selecciona una ciudad</option>
  <option value="bogota">Bogotá</option>
  <option value="medellin">Medellín</option>
</select>

<textarea name="mensaje" rows="5"
  placeholder="Escribe tu mensaje aquí..."></textarea>

<button type="submit">Enviar formulario</button>

Buenas prácticas:

  • Siempre especificar type en button (submit, button o reset).
  • Agregar una primera opción vacía en select como instrucción al usuario.
  • Controlar el tamaño de textarea con CSS en lugar de rows y cols.

5. Tablas

Las tablas en HTML permiten mostrar información organizada en filas y columnas. Son ideales para datos comparativos, horarios, calendarios y reportes. Es importante usarlas solo para datos tabulares y nunca para maquetar el diseño de una página, ya que para eso existe CSS.

<table>

Definición: Es el contenedor principal de una tabla. Dentro de él se colocan las filas, encabezados, cuerpo y pie de la tabla. Por sí sola no muestra nada; necesita las etiquetas tr, th y td para mostrar datos.

Sintaxis básica:

<table>
  <caption>Tabla de notas</caption>
  <thead> ... </thead>
  <tbody> ... </tbody>
  <tfoot> ... </tfoot>
</table>

Buenas prácticas:

  • Siempre agregar caption para describir el contenido de la tabla.
  • Nunca usar tables para maquetar páginas; usar CSS Grid o Flexbox.
  • Separar la tabla en thead, tbody y tfoot para mayor claridad semántica.

<tr>, <th> y <td>

Definición: tr define una fila dentro de la tabla. th define una celda de encabezado (se muestra en negrita y centrada por defecto, tiene significado semántico de cabecera). td define una celda de datos normal. Estas tres etiquetas trabajan siempre juntas para construir la estructura interna de la tabla.

Sintaxis básica:

<table>
  <thead>
    <tr>
      <th scope="col">Nombre</th>
      <th scope="col">Nota</th>
      <th scope="col">Estado</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana García</td>
      <td>4.5</td>
      <td>Aprobado</td>
    </tr>
    <tr>
      <td>Luis Pérez</td>
      <td>3.8</td>
      <td>Aprobado</td>
    </tr>
  </tbody>
</table>

Uso adecuado: Usar colspan para fusionar celdas horizontalmente y rowspan para fusionar celdas verticalmente cuando los datos lo requieran.

Buenas prácticas:

  • Usar scope="col" o scope="row" en th para mejorar la accesibilidad.
  • No aplicar estilos directamente en los atributos de td; usar clases CSS.

6. Multimedia

Las etiquetas multimedia permiten integrar imágenes, videos y audios directamente en la página sin necesidad de plugins externos. Desde HTML5, el navegador puede reproducir estos contenidos de forma nativa. Es importante usarlas correctamente para garantizar accesibilidad y buen rendimiento de carga.

<img>

Definición: Inserta una imagen en el documento. Es una etiqueta vacía, es decir, no tiene etiqueta de cierre. La imagen se carga desde la ruta indicada en el atributo src. El atributo alt es obligatorio y describe la imagen en texto para personas que usan lectores de pantalla o cuando la imagen no carga por algún error.

Sintaxis básica:

<img src="foto-perfil.jpg"
     alt="Fotografía de perfil del autor"
     width="300" height="300">

Uso adecuado: Para imágenes que forman parte del contenido. Si la imagen es solo decorativa (no aporta información), usar alt="" vacío para que los lectores de pantalla la ignoren.

Buenas prácticas:

  • Nunca omitir el atributo alt; su ausencia es un error de accesibilidad.
  • Especificar width y height para evitar que la página salte mientras carga.
  • Usar loading="lazy" en imágenes que están lejos del inicio de la página.
  • Usar formatos modernos como WebP para reducir el peso del archivo.

<video>

Definición: Incrusta un reproductor de video nativo directamente en la página. Admite múltiples fuentes con la etiqueta source para garantizar compatibilidad entre navegadores. El atributo controls muestra los botones de reproducción, pausa y volumen al usuario.

Sintaxis básica:

<video controls width="640" height="360">
  <source src="presentacion.mp4" type="video/mp4">
  <source src="presentacion.webm" type="video/webm">
  <p>Tu navegador no soporta la reproducción de video HTML5.</p>
</video>

Uso adecuado: El texto entre las etiquetas de video sirve de mensaje alternativo para navegadores muy antiguos que no soporten HTML5.

Buenas prácticas:

  • No usar autoplay con sonido; los navegadores modernos lo bloquean por defecto.
  • Agregar subtítulos con la etiqueta track para mejorar la accesibilidad.
  • Usar preload="metadata" para que el navegador solo cargue la información básica del video al inicio.

<audio>

Definición: Incrusta un reproductor de audio nativo en la página. Funciona de manera similar a video pero solo para contenido de sonido. También admite múltiples fuentes y el texto entre etiquetas sirve como alternativa para navegadores sin soporte.

Sintaxis básica:

<audio controls>
  <source src="podcast-episodio1.mp3" type="audio/mpeg">
  <source src="podcast-episodio1.ogg" type="audio/ogg">
  Tu navegador no soporta la reproducción de audio HTML5.
</audio>

Buenas prácticas:

  • Nunca reproducir audio automáticamente sin que el usuario lo solicite.
  • Incluir siempre el atributo controls para que el usuario pueda pausar o bajar el volumen.
  • Ofrecer al menos dos formatos (mp3 y ogg) para mayor compatibilidad entre navegadores.

<iframe>

Definición: Permite incrustar contenido externo dentro de la página, como un video de YouTube, un mapa de Google Maps o un documento PDF. El contenido se muestra en un marco independiente dentro de la página principal sin necesidad de redirigir al usuario a otro sitio.

Sintaxis básica:

<iframe
  src="https://www.youtube.com/embed/ID_DEL_VIDEO"
  width="560"
  height="315"
  title="Tutorial de HTML para principiantes"
  allowfullscreen>
</iframe>

Buenas prácticas:

  • Siempre incluir el atributo title con una descripción del contenido incrustado.
  • Usar el atributo sandbox para limitar los permisos del contenido externo.
  • No intentar incrustar páginas que no lo permitan; muchos sitios bloquean ser mostrados en iframe.
