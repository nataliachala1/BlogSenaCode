Blog Educativo: Etiquetas HTML por Categorías

Este blog es una guía educativa sobre las etiquetas más importantes de HTML. Está organizado en seis categorías: estructurales, texto, listas, formularios, tablas y multimedia. Para cada etiqueta encontrarás su definición, un ejemplo de uso y buenas prácticas recomendadas.

HTML (HyperText Markup Language) es el lenguaje base de todas las páginas web. Fue creado por Tim Berners-Lee en 1991 y desde entonces ha evolucionado hasta su versión actual, HTML5. Aprender sus etiquetas es el primer paso para construir sitios web.

1. Etiquetas Estructurales

Las etiquetas estructurales definen el esqueleto de una página web. Sin ellas, el navegador no sabría cómo interpretar el documento. Son las primeras que debes aprender porque forman la base de cualquier archivo HTML.

<html>

Definición: Es la etiqueta raíz de todo documento HTML. Todo el contenido de la página debe estar dentro de ella.

Sintaxis básica:

<html lang="es">
  ...
</html>

Uso adecuado: Debe aparecer justo después del DOCTYPE. Solo puede haber una por documento.

Buenas prácticas:

  • Siempre escribir el atributo lang con el idioma de la página.
  • No omitir el DOCTYPE antes de esta etiqueta.
<head>

Definición: Contiene la información del documento que no se muestra en pantalla: título, estilos, scripts y metadatos.

Sintaxis básica:

<head>
  <meta charset="UTF-8">
  <title>Título</title>
</head>

Buenas prácticas:

  • Incluir siempre meta charset="UTF-8".
  • Escribir un title descriptivo y único en cada página.
<body>

Definición: Contiene todo el contenido visible de la página: textos, imágenes, videos, tablas, formularios, etc.

Sintaxis básica:

<body>
  <h1>Bienvenido</h1>
  <p>Contenido de la página.</p>
</body>

Buenas prácticas:

  • Solo puede existir un body por documento.
  • No usar atributos visuales en línea; usar CSS.
<header>

Definición: Representa la cabecera de una página o sección. Contiene el logo, título principal y navegación.

Sintaxis básica:

<header>
  <h1>Mi Sitio Web</h1>
  <nav>...</nav>
</header>

Buenas prácticas:

  • No confundir con la etiqueta head.
  • No anidar un header dentro de otro header.
<main>

Definición: Delimita el contenido principal de la página. Solo debe haber uno por documento.

Sintaxis básica:

<main>
  <h2>Artículo principal</h2>
  <p>Contenido importante.</p>
</main>

Buenas prácticas:

  • Usar solo un main por página.
  • No meterlo dentro de header, footer o nav.
<footer>

Definición: Define el pie de página. Contiene derechos de autor, enlaces legales y datos de contacto.

Sintaxis básica:

<footer>
  <p>&copy; 2025 Mi Sitio.</p>
</footer>

Buenas prácticas:

  • Mantenerlo simple y con información relevante.
  • Se puede usar dentro de un article para indicar autor y fecha.

2. Etiquetas de Texto

Las etiquetas de texto permiten organizar y dar significado al contenido escrito. Indican al navegador y buscadores qué partes son más importantes, cuáles son títulos y cuáles son párrafos normales.

<h1> - <h6>

Definición: Etiquetas de encabezado que definen la jerarquía de títulos. h1 es el más importante y h6 el menos.

Sintaxis básica:

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

Buenas prácticas:

  • Usar solo un h1 por página.
  • No saltar niveles: no ir de h1 directamente a h4.
<p>

Definición: Define un párrafo de texto. Es la etiqueta de texto más usada en HTML.

Sintaxis básica:

<p>Este es un párrafo de ejemplo.</p>

Buenas prácticas:

  • No usar br varias veces para separar textos; usar p.
  • No meter etiquetas de bloque dentro de un p.
<strong> y <em>

Definición: strong marca texto importante (negrita). em marca texto con énfasis (cursiva). Ambas tienen significado semántico real.

Sintaxis básica:

<p>Es <strong>obligatorio</strong> guardar
antes de <em>cerrar el editor</em>.</p>

Buenas prácticas:

  • No usarlas solo para cambiar el aspecto visual.
  • No abusar: si todo está marcado, nada es importante.
<span>

Definición: Contenedor en línea sin significado semántico. Se usa para aplicar estilos CSS a partes del texto.

Sintaxis básica:

<p>El descuento es del
<span class="resaltado">30%</span> hoy.</p>

Buenas prácticas:

  • Preferir etiquetas semánticas cuando aplique.
  • Siempre usar junto con una clase o id.

3. Listas

Las listas permiten organizar información en grupos de elementos relacionados. Son muy usadas en menús, instrucciones paso a paso y glosarios.

<ul> - Lista desordenada

Definición: Crea una lista de elementos sin orden específico. Cada elemento se muestra con una viñeta.

Sintaxis básica:

<ul>
  <li>Manzana</li>
  <li>Naranja</li>
</ul>

Buenas prácticas:

  • El único hijo directo válido de ul es li.
  • Cambiar el estilo de las viñetas con CSS.
<ol> - Lista ordenada

Definición: Crea una lista numerada automáticamente. El orden de los elementos es importante.

Sintaxis básica:

<ol>
  <li>Abrir el navegador</li>
  <li>Escribir la dirección</li>
  <li>Presionar Enter</li>
</ol>

Buenas prácticas:

  • Usar start para comenzar desde un número diferente al 1.
  • Usar reversed para orden descendente.
<dl>, <dt>, <dd> - Lista de definición

Definición: dl agrupa pares término-descripción. dt es el término y dd es su descripción.

Sintaxis básica:

<dl>
  <dt>HTML</dt>
  <dd>Lenguaje para crear páginas web.</dd>
  <dt>CSS</dt>
  <dd>Lenguaje para dar estilo visual.</dd>
</dl>

Buenas prácticas:

  • Un dt puede tener varios dd asociados.
  • No usar dl solo para crear sangría visual.

4. Formularios

Los formularios permiten recopilar información del usuario. Son fundamentales en páginas de registro, contacto y compras en línea.

<form>

Definición: Contenedor principal de un formulario. Define hacia dónde se envían los datos y el método de envío.

Sintaxis básica:

<form action="/enviar" method="post">
  <!-- campos aquí -->
</form>

Buenas prácticas:

  • Usar method="post" para datos privados como contraseñas.
  • No anidar un form dentro de otro form.
<input>

Definición: Campo de entrada más versátil. Cambia según el atributo type: texto, contraseña, email, checkbox, etc.

Sintaxis básica:

<input type="text" name="nombre" placeholder="Tu nombre" required>
<input type="email" name="correo">
<input type="password" name="clave">
<input type="checkbox" name="terminos">

Buenas prácticas:

  • Siempre incluir el atributo name.
  • Usar required en campos obligatorios.
<label>

Definición: Etiqueta descriptiva de un campo. Al hacer clic en ella, el cursor se mueve al campo asociado.

Sintaxis básica:

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre">

Buenas prácticas:

  • El valor de for debe ser igual al id del input.
  • Nunca omitir el label.
<select>, <textarea> y <button>

Definición: select crea un menú desplegable. textarea permite texto multilínea. button crea un botón interactivo.

Sintaxis básica:

<select name="ciudad">
  <option value="">Selecciona una ciudad</option>
  <option value="bogota">Bogotá</option>
</select>

<textarea name="mensaje" rows="5"
  placeholder="Escribe aquí..."></textarea>

<button type="submit">Enviar</button>

Buenas prácticas:

  • Siempre especificar type en button.
  • Agregar una primera opción vacía en select.

5. Tablas

Las tablas permiten mostrar información en filas y columnas. Son ideales para datos comparativos, horarios y reportes. Nunca deben usarse para maquetar páginas.

<table>

Definición: Contenedor principal de una tabla. Necesita tr, th y td para mostrar datos.

Sintaxis básica:

<table>
  <caption>Tabla de notas</caption>
  <thead> ... </thead>
  <tbody> ... </tbody>
</table>

Buenas prácticas:

  • Siempre agregar caption para describir la tabla.
  • Nunca usar tables para maquetar páginas.
<tr>, <th> y <td>

Definición: tr define una fila. th define una celda de encabezado. td define una celda de datos normal.

Sintaxis básica:

<table>
  <thead>
    <tr>
      <th scope="col">Nombre</th>
      <th scope="col">Nota</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana García</td>
      <td>4.5</td>
    </tr>
  </tbody>
</table>

Buenas prácticas:

  • Usar scope="col" o scope="row" en th para accesibilidad.
  • No aplicar estilos directamente en td; usar clases CSS.

6. Multimedia

Las etiquetas multimedia permiten integrar imágenes, videos y audios sin plugins externos. Desde HTML5 el navegador los reproduce de forma nativa.

<img>

Definición: Inserta una imagen. Es una etiqueta vacía; la imagen se carga desde el atributo src.

Sintaxis básica:

<img src="foto.jpg"
     alt="Descripción de la imagen"
     width="300" height="300">

Buenas prácticas:

  • Nunca omitir el atributo alt.
  • Especificar width y height para evitar saltos al cargar.
  • Usar loading="lazy" en imágenes lejos del inicio.
<video>

Definición: Incrusta un reproductor de video nativo. Admite múltiples fuentes para compatibilidad entre navegadores.

Sintaxis básica:

<video controls width="640" height="360">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <p>Tu navegador no soporta video HTML5.</p>
</video>

Buenas prácticas:

  • No usar autoplay con sonido.
  • Agregar subtítulos con la etiqueta track.
<audio>

Definición: Incrusta un reproductor de audio nativo en la página.

Sintaxis básica:

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Tu navegador no soporta audio HTML5.
</audio>

Buenas prácticas:

  • Nunca reproducir audio automáticamente.
  • Incluir siempre el atributo controls.
<iframe>

Definición: Incrusta contenido externo como videos de YouTube o mapas de Google Maps en un marco independiente.

Sintaxis básica:

<iframe
  src="https://www.youtube.com/embed/ID"
  width="560" height="315"
  title="Descripción del video"
  allowfullscreen>
</iframe>

Buenas prácticas:

  • Siempre incluir el atributo title.
  • Usar sandbox para limitar permisos del contenido externo.
