Etiquetas de Formularios

Los formularios permiten recopilar información del usuario. Son fundamentales en páginas de registro, contacto y compras en línea. Un formulario bien construido mejora la experiencia del usuario y facilita el procesamiento de datos.

<form>

Definición: Contenedor principal de un formulario. Define hacia dónde se envían los datos (atributo action) y el método de envío (atributo method: get o post).

Sintaxis básica:

<form action="/enviar" method="post">
  <!-- campos aquí -->
</form>

Uso adecuado: Usar method="post" para datos privados. Usar method="get" solo para búsquedas donde los datos pueden aparecer en la URL.

Buenas prácticas:

  • Usar method="post" para datos privados como contraseñas.
  • No anidar un form dentro de otro form.
  • Siempre asociar cada campo con un label.
<input>

Definición: Campo de entrada más versátil del formulario. Cambia completamente su comportamiento según el valor del atributo type: texto, contraseña, email, checkbox, radio, fecha, archivo, etc.

Sintaxis básica:

<input type="text" name="nombre" placeholder="Tu nombre" required>
<input type="email" name="correo">
<input type="password" name="clave">
<input type="checkbox" name="terminos">

Uso adecuado: Elegir el type correcto activa validaciones automáticas del navegador. Por ejemplo type="email" valida el formato de correo.

Buenas prácticas:

  • Siempre incluir el atributo name para que el dato llegue al servidor.
  • Usar required en campos obligatorios.
  • Vincular siempre con un label mediante for e id.
<label>

Definición: Etiqueta descriptiva de un campo. Indica al usuario qué información debe escribir. Al hacer clic en el label, el cursor se mueve automáticamente al campo asociado.

Sintaxis básica:

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre">

Uso adecuado: Fundamental para accesibilidad. Los lectores de pantalla anuncian el label cuando el usuario enfoca el campo.

Buenas prácticas:

  • El valor de for debe ser igual al id del input.
  • Nunca omitir el label; si no debe verse, ocultarlo con CSS.
<select>, <textarea> y <button>

Definición: select crea un menú desplegable de opciones. textarea permite escribir texto en múltiples líneas. button crea un botón que puede enviar el formulario, limpiarlo o ejecutar una acción con JavaScript.

Sintaxis básica:

<select name="ciudad">
  <option value="">Selecciona una ciudad</option>
  <option value="bogota">Bogotá</option>
  <option value="medellin">Medellín</option>
</select>

<textarea name="mensaje" rows="5"
  placeholder="Escribe aquí..."></textarea>

<button type="submit">Enviar</button>

Uso adecuado: button type="submit" envía el formulario. button type="button" no lo hace (usado con JS). button type="reset" limpia los campos.

Buenas prácticas:

  • Siempre especificar type en button para evitar comportamientos inesperados.
  • Agregar una primera opción vacía en select como instrucción al usuario.
  • Controlar el tamaño de textarea con CSS en lugar de rows y cols.
